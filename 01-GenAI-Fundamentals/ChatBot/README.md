# 🧠 Concepts Learned

---

## 1. Conversation History

The chatbot maintains a conversation using a single array.

```js
messages = [
    ...
]
```

Every message exchanged between the user and assistant is stored here.

Example:

```
System

↓

User

↓

Assistant

↓

User

↓

Assistant
```

This array becomes the chatbot's working memory.

---

## 2. Roles

Every message has a role.

### System

Provides instructions to the LLM.

Example:

```
You are a helpful assistant.
```

---

### User

Represents messages sent by the user.

Example:

```
Hi
```

---

### Assistant

Represents responses generated by the LLM.

Example:

```
Hello! How can I help you today?
```

---

## 3. Stateless LLM

One of the most important concepts.

Large Language Models do **NOT** remember previous conversations.

Every API request is independent.

The backend creates the illusion of memory by sending the complete conversation every time.

```
Backend

↓

messages[]

↓

LLM

↓

Response
```

The memory belongs to the application, **not the model**.

---

## 4. Why We Store Both User and Assistant Messages

Without assistant messages:

```
User

↓

User

↓

User
```

The model would not know what it previously answered.

Without user messages:

```
Assistant

↓

Assistant

↓

Assistant
```

The model would not know what questions were asked.

A conversation requires both sides.

---

## 5. Conversation Loop

The chatbot continuously follows this cycle:

```
User Input

↓

Store User Message

↓

Send messages[]

↓

LLM

↓

Assistant Reply

↓

Store Assistant Reply

↓

Display Response

↓

Repeat
```

---

## 6. System Prompt

The System Prompt defines the behaviour of the chatbot.

Example:

```
You are a helpful AI assistant.
```

It is always the first message inside the conversation.

---

## 7. Context Window (Introduction)

Every LLM has a maximum amount of information it can process in a single request.

This is called the **Context Window**.

Instead of thinking only about messages:

```
messages[]
```

Think about:

```
messages[]

↓

Tokens

↓

Context Window
```

The longer the conversation becomes, the more tokens are consumed.

---

## 8. Tokens

LLMs don't process words directly.

They process **tokens**.

More conversation

↓

More tokens

↓

Higher cost

↓

Slower requests

↓

Eventually reaching the model's context limit.

# 💡 Engineering Notes

## Why does ChatGPT remember previous messages?

It doesn't.

The backend stores every message and sends the entire conversation again.

---

## Why keep the System Prompt at the beginning?

Because it defines the assistant's behaviour before the conversation starts.

It provides high-level instructions that guide every response.

---

## Why push assistant responses into the message array?

Without them, the model would forget its own previous answers.

The assistant also needs conversational history.

---

## Why not send only the latest user message?

Because the model would lose all previous context.

Every request would become an entirely new conversation.

---

## Can we keep sending messages forever?

No.

Problems:

- Higher API cost
- Slower responses
- Context window limits
- Large prompts

---

## How do production systems solve this?

Instead of sending the complete history:

```
System Prompt

+

Recent Messages

+

Conversation Summary

+

Relevant Facts

+

Current User Message
```

Everything else is stored in a database.

# 📌 Personal Note

This project helped me understand that ChatGPT does not "remember" conversations.

Instead, the backend continuously reconstructs the conversation by sending the relevant history to the model.

This shifted my thinking from:

> "The AI remembers."

to

> "The backend manages memory, and the LLM reasons over the context it receives."

This is one of the fundamental concepts behind modern AI systems and will be essential when building **ScrollDamage AI Content Operating System**.
